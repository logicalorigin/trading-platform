## Latest Continuation Update - 2026-06-02 15:11 MDT

### Root Cause

- Algo's top-level screen module was preloadable, but the actual first-viewport operations body was still behind nested dynamic imports.
- `AlgoScreen` loaded `AlgoLivePage` only on visible render, and `AlgoLivePage` then lazily loaded `OperationsSignalTable` and `OperationsPositionsTable`.
- Those table chunks pulled the heavy first-viewport dependencies (`OperationsSignalRow` and `PositionsPanel`) after the live-page chunk, producing a client-side module waterfall.
- The right rail also loaded immediately in visible startup before the main operations table had settled. It imports `AlgoRightRail`, `AlgoSettingsRegion`, and `algoSettingsFields`; in dev this could consume request/compile attention and delay visible operations containers.
- A separate setup-data gate made the page return `EmptyOperationsState` any time deployments were not present yet. That meant slow setup queries could hide the operations shell even when the visible page should have rendered an empty-but-stable shell while data settled.

### Implemented Fix

- Added a generic nested preload hook in `artifacts/pyrus/src/features/platform/screenModulePreloader.js`:
  - When any loaded screen module exports `preloadScreenModules`, the platform preloader calls it opportunistically.
  - Errors are intentionally swallowed because visible route render still retries the real import.
- Added `preloadScreenModules` to `artifacts/pyrus/src/screens/AlgoScreen.jsx`:
  - Preloads `AlgoLivePage`.
  - Preserves visible-screen query ownership; hidden mounted Algo still does not start its data queries.
  - Does not preload the right rail in the nested preload block.
- Changed `artifacts/pyrus/src/screens/algo/AlgoLivePage.jsx`:
  - `OperationsSignalTable` and `OperationsPositionsTable` are now static imports because they are first-viewport containers, not below-fold decoration.
  - Removed the lazy table wrappers and `AlgoDeferredBlock` table fallbacks for those containers.
  - Added `rightRailFallback` plus a local `ALGO_RIGHT_RAIL_DEFER_MS = 1_500` gate so right-rail code starts after the main operations body can commit.
  - Changed empty-state gating to `setupDataSettled && !deployments.length`, so pending setup data no longer hides the operations shell.
- Updated targeted source guards and render tests:
  - `artifacts/pyrus/src/features/platform/platformRootSource.test.js`
