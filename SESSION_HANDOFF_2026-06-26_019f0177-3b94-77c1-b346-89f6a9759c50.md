## Observed Facts

- Backend option chains and expirations now use Massive option-chain and historical option-contract APIs.
- Option quote snapshots and the compatibility option quote stream facade now hydrate from Massive OPRA snapshots, not broker option quote snapshots or broker streaming.
- Option chart bars now use Massive option aggregates. The broker live-edge option history merge path was removed.
- Option flow scanner, GEX, account option quote hydration, shadow-account quote hydration, signal option tick handling, and the related frontend labels were updated to Massive/OPRA wording and behavior.
- The options flow scanner now gates on Massive option data availability, not broker/TWS health.
- IBKR pressure events no longer shed or damp Massive flow-scanner leases.
- Option market-depth calls are explicitly unavailable instead of falling through to broker depth.
- Durable option metadata now separates OPRA market-data identifiers from broker contract identifiers with `broker_contract_id`.
- OpenAPI and generated API clients were regenerated after schema changes.
- The repo was already heavily dirty before this work. Unrelated modified and untracked files were left untouched.
