## Scope

User reported that shadow SPY option rows had bid/ask/last, while real F option rows stayed pending. User also reported connection popover drift in the `Free` column rows.

## Observed Facts

- Real account endpoint before the source fix returned F `880754762` with bid/ask/last `null`, quote status `pending`, reason `awaiting_quote`.
- Shadow endpoint returned SPY option bid/ask/last from the shadow direct snapshot path.
- IBKR line usage showed F admitted as `account-monitor-live`, pool `account-monitor`, priority `90`, line `option:880754762`.
- Option quote stream diagnostics excluded `880754762` from `desiredProviderContractIds` despite the account-monitor live lease.
- Direct snapshot probe against `/api/options/quotes` for F `880754762` returned bid `0.59`, ask `0.61`, freshness `stale`, market data mode `frozen`.
- After that direct snapshot populated cache in the already-running API process, real positions returned F bid `0.59`, ask `0.61`, last `0.57`, mark `0.60`, quote status `live`, market data mode `frozen`.
- Connection popover renderer was special-casing `account-monitor`: `Available` displayed `row.needed`, and `Free` displayed `row.deferred`, while other rows displayed cap/free.

## Root Cause

- Real account positions declared account-position option quote demand, then immediately read demand/cache state. Unlike shadow positions, they did not perform a bounded direct bridge option snapshot fetch before the read.
- F was already admitted at account-monitor priority, but the quote cache was not hydrated for the real row until another path explicitly fetched the option snapshot.
- The connection popover column drift was a frontend display bug: the `Free` column did not mean `row.free` for the account-monitor row.

## Changes

- `artifacts/api-server/src/services/account.ts`
  - Added bounded per-underlying account position option snapshot hydration via `fetchBridgeOptionQuoteSnapshots`.
  - Uses intent `account-monitor-live`, fallback `cache`, and `requiresGreeks: true`.
  - Waits up to `ACCOUNT_OPTION_QUOTE_SNAPSHOT_TASK_MAX_WAIT_MS = 3000` before reading demand state.
- `artifacts/pyrus/src/features/platform/HeaderStatusCluster.jsx`
  - Removed the account-monitor special-case in line usage rows.
  - `Available` now uses `row.effectiveCap ?? row.cap` for every row.
  - `Free` now uses `row.free` for every row.
