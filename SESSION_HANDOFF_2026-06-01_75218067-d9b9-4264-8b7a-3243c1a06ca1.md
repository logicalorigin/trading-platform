## Latest Update: Definitive Massive WebSocket Churn Cause

- User asked for definitive cause rather than a likely cause.
- Direct Massive WebSocket probe using the same API key and the live 92-symbol universe captured Massive's explicit server response:
  - `status: "max_connections"`
  - `message: "Maximum number of websocket connections exceeded. You have reached the connection limit for your account. Please contact support at https://massive.com/contact to increase your limit."`
  - WebSocket close code: `1008`
- The AM-only probe authenticated, subscribed to all requested `AM.*` symbols, then received `max_connections` and closed after about 4.8s.
- A combined single-socket `AM/Q/T` probe also received the same `max_connections` status and `1008` close while the app was running, proving the failure is an account-level Massive connection limit rather than a symbol payload/field parsing issue.
- App architecture evidence:
  - `artifacts/api-server/src/services/massive-stock-aggregate-stream.ts` owns its own process-level `new WebSocket(...)` for `AM.*` minute aggregates.
  - `artifacts/api-server/src/services/massive-stock-quote-stream.ts` owns a separate process-level `new WebSocket("wss://socket.massive.com/stocks")` for `Q.*` quotes and `T.*` trades.
  - Current diagnostics expose reconnect counts but do not record non-auth Massive status messages, close codes, or close reasons, so the app was hiding the exact `max_connections` close reason.
- Definitive cause: the running workspace is exceeding the Massive account's concurrent WebSocket connection limit. The app's split stock WebSocket clients consume multiple stock connections before any external probes/browser/runtime duplicates are counted, then Massive closes sockets with `max_connections`, which creates the observed reconnect churn and stale stream windows.
- First fix: replace the split aggregate and quote/trade clients with one shared process-level Massive stocks WebSocket manager that unions `AM`, `Q`, and `T` subscriptions, fans messages out to both existing consumers, and records Massive status/close diagnostics.
