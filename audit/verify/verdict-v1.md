HEAD advanced from `38513ebf` during the audit, but the cited implementation files had no committed or working-tree diff across that interval. ## Verdicts

P0-1: REAL — `/universe/logo-proxy` accepts every path on the shared `storage.googleapis.com` host, mirrors the upstream `Content-Type`, and returns the bytes without CSP, `Content-Disposition`, or `nosniff`; `/universe` misses the only path auth gate. `artifacts/api-server/src/routes/platform.ts:272-278,2390-2427`; `artifacts/api-server/src/routes/index.ts:44-89`

Concrete actor/victim trace: attacker A publishes a public object as `text/html` or `image/svg+xml` at `https://storage.googleapis.com/<A-bucket>/<object>` and sends signed-in user B `/api/universe/logo-proxy?url=<encoded-object-url>`. The initial-host check passes, PYRUS returns A's active bytes under the PYRUS URL and media type, and B's browser executes them with the PYRUS origin. The code can then call `/api/auth/session` to obtain B's CSRF token and make B-authenticated same-origin API requests while the document is open. `artifacts/api-server/src/routes/auth.ts:120-125,255-264`

Decisive checks:

- The allowlist comparison is an exact initial-host check, so hostname-suffix tricks are refuted. It is still insufficient because `storage.googleapis.com` is a multi-tenant path-style endpoint and neither bucket nor object path is constrained. Google documents both `https://storage.googleapis.com/BUCKET/OBJECT` public access and uploader-controlled object `Content-Type`: [public object URLs](https://docs.cloud.google.com/storage/docs/access-public-data), [object metadata](https://docs.cloud.google.com/storage/docs/metadata).
- `sameOriginLogoUrl` only rewrites URLs returned by `/universe/logos`; direct proxy requests reread raw `req.query.url`. `artifacts/api-server/src/routes/platform.ts:281-293,2368-2394`
- `fetch` has no redirect policy and the final `upstream.url` is never revalidated. A local current-runtime check observed the default follow behavior across origins, consistent with the [Fetch Standard](https://fetch.spec.whatwg.org/). This is a redirect-SSRF sink, although repo-only evidence does not prove an attacker-controlled redirect primitive on one of the allowed hosts. The direct GCS active-content path is sufficient for the REAL verdict. `artifacts/api-server/src/routes/platform.ts:2415-2418`
- The only route response protections are public caching and CORP before the upstream media type and fully buffered body are sent. The two-second timeout bounds time, not bytes. Global middleware adds Reporting/COOP/COEP/CORP only. `artifacts/api-server/src/routes/platform.ts:2423-2427`; `artifacts/api-server/src/app.ts:48-69`
- A disposable intercepted Chromium navigation reproduced the route/global response headers at `http://127.0.0.1:8080`. Both `text/html` and `image/svg+xml` set a DOM execution marker equal to `http://127.0.0.1:8080`. This confirms direct-navigation execution; SVG embedded only through `<img>` would be passive. Production mounts `/api` and the built web app on the same Express origin. `artifacts/api-server/src/app.ts:262-278`; `artifacts/pyrus/.replit-artifact/artifact.toml:28-34`

Minimal fix: allow only HTTPS and provider-owned bucket/path prefixes, reject redirects or revalidate every hop, and accept only magic-byte-verified raster formats. Stream through a small byte cap and set a server-owned image MIME plus `nosniff`; do not serve HTML/SVG as an inline same-origin document.

P1-8: REAL — anonymous `/api/flow/scanner/benchmark` can register process-global, five-minute Massive quote demand whose poller launches one REST snapshot request per admitted contract after every at-least-two-second interval; no auth or request rate limiter covers the path. `artifacts/api-server/src/routes/index.ts:39-89`; `artifacts/api-server/src/services/platform.ts:17666-17714`; `artifacts/api-server/src/services/bridge-option-quote-stream.ts:1021-1057,1377-1425`

Concrete two-user/shared-capacity trace: anonymous user B sends `POST /api/flow/scanner/benchmark` with `{"underlying":"SPY","lineBudgets":[150],"expirationScanCount":1,"strikeCoverage":"full"}` and no cookie. Because `/flow` falls through the auth gate, PYRUS uses its shared Massive credential, admits up to the available flow-scanner cap (100 contracts by default), and retains B-created demand for five minutes. The process-global poller then repeatedly spends provider/API capacity shared by signed-in user A; B can repeat with cold optionable symbols because no per-IP or per-principal limiter exists.

Decisive checks and caps:

- `/api` is the mount, the handler contains no `requireUser`/`requireAdmin`, and a missing session explicitly continues through global context middleware. `artifacts/api-server/src/app.ts:220-236,262`; `artifacts/api-server/src/routes/platform.ts:2851-2901`
- Route admission classifies `/flow/*` as deferred analytics and permits it under normal hard-resource pressure. High-pressure/QA-safe shedding is a conditional circuit breaker, not authentication or rate limiting. `artifacts/api-server/src/services/route-admission.ts:207-227,387-395,471-545`
- Positive budgets are floored, clamped to `1..150`, deduplicated, and then run sequentially. Therefore one request has at most 150 unique budget passes; the amplification is finite, not unbounded. `artifacts/api-server/src/services/platform.ts:19009-19020,19101-19295`
- The cited “broker data” wording is stale. Current expiration, chain, bar, and quote work resolves to Massive. Metadata expiration/chain calls explicitly set `bypassBridgeBackoff: true`, but the retained live poll does have transient-error backoff; its success path has no provider limiter. `artifacts/api-server/src/services/platform.ts:15112-15128,15153-15178,19133-19168`; `artifacts/api-server/src/services/bridge-option-quote-stream.ts:1047-1057,1377-1425`
- Live candidates are capped by the requested budget and then by shared admission capacity. The default scanner line cap is 100, quote-demand TTL is 300,000 ms, and benchmark hydration deliberately does not release leases on completion or abort. The stream chunks at 100 contracts and uses `Promise.all` to make one Massive contract-snapshot request per contract each poll. With free default capacity and negligible provider latency, this is nominally on the order of 15,000 requests over five minutes; actual calls are lower with latency, occupied capacity, or provider backoff. `artifacts/api-server/src/services/platform.ts:11587-11619,11939-11943,17274-17288,17704-17714`; `artifacts/api-server/src/services/bridge-option-quote-stream.ts:168-172,1404-1425`
- Historical hydration is separately capped to eight candidates per budget by default and concurrency two, but it performs uncached Massive option-bar requests. `expirationScanCount: 0` selects all discovered expirations, while the batch emergency expiration ceiling defaults disabled. Client disconnect does not cancel benchmark work because the route passes no request abort signal. `artifacts/api-server/src/services/platform.ts:11503-11507,11641-11647,16425-16435,17175-17205,17401-17408,17551-17614,2893-2900`
- No repository middleware provides a per-route/per-IP limiter here. Any external edge/WAF limit is unknown and is not a source-backed protection.

Minimal fix: make the exact benchmark path admin-only or unavailable in production, and add strict per-principal/IP concurrency plus rate limits. Bound it to one/few budgets and a small positive expiration count, honor metadata backoff, pass the request abort signal, and release benchmark quote leases when the request completes.

P1-7: PARTIAL — the tracked throwaway script really inserts `role: "admin"` and prints a fresh 12-hour bearer token, but no build, start, deploy, package-script, or CI path executes it; production bundles only `src/index.ts`. `artifacts/api-server/src/scripts/.qa-chooser-session.mts:7-13`; `artifacts/api-server/build.mjs:18-23`

Why this is not REAL credential exposure on current evidence:

- Git tracks the file, and it has no `NODE_ENV` or production guard. If an operator explicitly runs it with production database credentials, it will create an admin user and emit the only raw session token to stdout. `tsx` is installed as a development dependency. `artifacts/api-server/src/scripts/.qa-chooser-session.mts:1-13`; `artifacts/api-server/package.json:29-40`
- The default session lifetime is 12 hours; the database stores the token hash, not the raw token. A holder can set the session cookie, call `/api/auth/session` for CSRF, and use admin bypasses, including unfiltered cross-tenant algo deployment reads. `artifacts/api-server/src/services/auth.ts:22-24,194-232`; `artifacts/api-server/src/services/automation-authorization.ts:21-30,98-103`
- The production build has one entry point, `src/index.ts`, and production runs only `artifacts/api-server/dist/index.mjs`. The QA source is not imported, bundled, HTTP-reachable, or named by any tracked package/CI hook. `artifacts/api-server/build.mjs:18-23`; `artifacts/pyrus/.replit-artifact/artifact.toml:22-34`; `artifacts/api-server/package.json:6-10`
- A tracked companion file confirms manual developer/agent QA usage: it reads a token from a session-specific `/tmp/.../scratchpad/qa-session.json` and installs it into a disposable browser context. That is not an automatic production or CI execution channel. `artifacts/pyrus/e2e/.qa-chooser-shot.mjs:1-17`

Conditional trace only: a privileged runner explicitly executes the source against a target database, a separate actor can read that stdout/scratch output, and that actor reuses the token as admin for up to 12 hours. The repository proves the dangerous manual capability but not an execution or lower-privilege log-reader path, so PARTIAL is the strongest supported verdict.

## Verification posture

Three independent fresh-context reviews tried to refute the findings, followed by source reconciliation. P0-1's redirect-SSRF subclaim remains only a sink without a proven redirect primitive, but its same-origin active-content path was browser-confirmed. P1-8 is capped and targets Massive rather than IBKR, but the anonymous retained polling is still real. P1-7 has no automatic execution path. Cross-model CLI review was offered but not run because no explicit authorization was provided.
