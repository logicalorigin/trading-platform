## Root Cause Fixed

Observed issue: `/api/bars` could return successful but cold/stale/partial historical responses for chart-oriented hydration. The backtest worker treated those responses as complete, persisted bad historical datasets, and produced missing-return samples.

Implemented fix:

- Worker bar requests now pass `requireFreshHistorical=true`, `allowHistoricalSynthesis=true`, `fetchPriority=8`, `requestFamily=backtest-worker`, and `brokerRecentWindowMinutes=30`.
- API route/schema/service now accepts `requireFreshHistorical` and keeps that mode out of reusable fresh/stale cache reads when the response is incomplete.
- API does not cache incomplete cold/partial/warming required-fresh-historical responses.
- Worker retries incomplete fresh historical responses, checks newest returned bar coverage, and errors rather than persisting incomplete history.
- Worker segmented historical loads now align non-final segment ends to UTC day end and advance by 1ms.
- Historical bar inserts now chunk at 10 rows to avoid statement timeout failures.
